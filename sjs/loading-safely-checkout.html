Loading Safely Checkout

After initialization of the Safely object, you can begin making calls to the methods available. Each of these methods is documented below, along with the accepted parameters.

Methods​

The Safely object gives access to methods used to initialize and display checkout options with Safely Finance on your website. The following methods are available after initializing safely.js.

render()​

Rendering the checkout is necessary if you'd like to have Safely display payment options on your website, as seen in the image below (the exact look and feel may vary).

In order to render this checkout, you must supply a containing element in which safely.js can render the checkout options. Anything inside of this element will be replaced upon calling render on the safely.js object.

Syntax​

safely.render(selector, options);

Parameters​

selector

The selector of the element where Safely Checkout should be rendered

options

Any options to pass in to the render call

Example​

Sample Element
<div id="safely_checkout"></div>
Loading and rendering the Safely Checkout
safely.render('#safely_checkout', {amount: '5200.00'});

loadCheckout()​

Using the checkout method directly, you can render your own payment options on your website, and only trigger the Safely Checkout once the user has selected one.

Syntax​

safely.loadCheckout(options);

Parameters​

options

The selector of the element where Safely Checkout should be rendered

options

charges - An array of charge objects for the customer
successUrl - A URL to redirect the user to after a successful checkout
cancelUrl - A URL to redirect the user to if they abandon or fail a checkout
metadata - Any reference data you'd like to provide that will be sent to callback URLs

Example​

Loading the checkout screen
safely.loadCheckout({
  successUrl: 'https://safelyfinance.co/success',
  cancelUrl: 'https://safelyfinance.co/error',
  metadata: {
    customerRef: '412414'
  },
  charges: [
    {amount: '1200.00', description: 'moving truck'},
    {amount: '400.00', description: 'self storage'}
  ]
});

Callbacks​

Some methods on safely.js allow for callback URLs to be provided. These URLs are used by safely.js to return to the user to a previous or new state after processing.

An example of this may be a successUrl for sending a user back to a confirmation page on your website after they've succesfully completed a checkout with Safely Finance.

Result handling​

The safely.js object returns a Promise that can be resolved to retrieve the results of the call, when available. The promise can be used as follows.

safely.checkout(/*...*/).then(function(result) {
  // Read the result of the call
});

Error handling​

The safely.js Promise also resolves error results that can be read (if an error occurred).

Checking for an error
safely.checkout(/*...*/).then(function(result) {
  // Check for an error in the result
  if (result.error) {
    // Handle the error
  }

  // No error, process the result
});
